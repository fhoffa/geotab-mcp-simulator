Connect your real Geotab account

Three things to do before an AI assistant touches your real fleet data.

  1. Get access to a MyGeotab database. The fastest way to start is a free demo database — register at my.geotab.com/registration.html to practice on anonymized data first (same shape as this simulator, no real drivers or vehicles). For production access, ask your Geotab account rep or fleet admin.
  2. Connect the Geotab MCP connector to your assistant of choice. Geotab publishes an official MCP server. Claude is the clearest starting point today; Microsoft Copilot, ChatGPT, and other MCP-capable clients can connect to the same server using the same URL below; only the menu names differ.

    Start with Geotab's official walkthrough: Getting started with MyGeotab MCP. The quick version is below.

    Example client setup:

    1. Go to claude.ai → Settings → Connectors, then click Add custom connector.
    2. Fill in a name (e.g. "Geotab") and the remote MCP server URL: https://mcp.geotab.com/mygeotab. Leave the OAuth fields blank — Geotab's server handles that for you.
    4. Click Add — Claude redirects you to a MyGeotab sign-in screen. Log in with your MyGeotab username and password and approve the connection.
    5. Back in the client, the connector now shows as Connected. Turn it on for a chat from the tools/connectors picker (the connector icon below the message box), then ask it anything you tried in this simulator.

    Microsoft Copilot, ChatGPT, and other MCP-capable clients follow the same pattern — add a custom/remote MCP connector, point it at https://mcp.geotab.com/mygeotab, and sign in with your MyGeotab credentials when prompted.

  3. Know what you're exposing.

    Before connecting a production database: the connector inherits your MyGeotab permissions, so the assistant can see anything you can see — including personal data (driver names, employee numbers, emails, precise location history).

    • Review your AI provider's DPA and your GDPR / data-minimization obligations.
    • Instruct the assistant to work at the vehicle/asset level — ask it to avoid driver names, emails, and phone numbers in its answers.
    • Demo databases (like the one above) are anonymized — "Demo - 01", no real driver data — so they're safe to explore freely.

    Not legal advice — check with your own counsel/compliance team before connecting production data.
